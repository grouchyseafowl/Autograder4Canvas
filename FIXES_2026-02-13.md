# Autograder Fixes Applied - February 13, 2026

## Problem Summary
The automated autograder had not been running successfully since February 4, 2026. Investigation revealed:
- A hung process from Feb 6 still running (7+ days)
- Missing timeouts on Canvas API requests causing indefinite hangs
- n8n notification service not running (non-critical but logged errors)
- No safeguards against long-running or stuck processes

## Root Causes Identified

### 1. Missing Request Timeouts
**Location**: `src/Programs/Autograder_Complete-Incomplete_v1-3.py`

Multiple Canvas API requests had no timeout parameter:
- `get_active_students()` - could hang forever waiting for student data
- `get_all_assignments()` - could hang fetching assignment list
- `get_submissions()` pagination loop - could hang on any page
- `get_assignment_name()` - could hang on single assignment
- Grade submission POST - could hang submitting grades
- Progress polling GET - could hang checking job status
- Course verification - could hang on initial API check

**Impact**: If Canvas API was slow or unresponsive, requests would wait indefinitely.

### 2. Infinite Polling Loop
**Location**: `src/Programs/Autograder_Complete-Incomplete_v1-3.py`, line 513

The grade job status polling loop had no timeout:
```python
while True:
    prog_resp = requests.get(progress_url, headers=HEADERS)
    # ... check status ...
    time.sleep(3)
```

If Canvas job stayed in "queued" or "running" state indefinitely, loop would never exit.

### 3. Unsafe Exception Handling
**Location**: `src/automation/canvas_helpers.py`, line 69

Bare `except:` clause could catch and suppress critical errors:
```python
except:  # Catches everything, including KeyboardInterrupt
    break
```

### 4. No Process Watchdog
No mechanism to detect and kill hung processes automatically.

## Fixes Applied

### 1. Request Timeout Additions ✅

**File**: `src/Programs/Autograder_Complete-Incomplete_v1-3.py`

Added 30-second timeouts to all Canvas API requests:
- Line 196: `get_active_students()` - timeout=30
- Line 215: `get_all_assignments()` - timeout=30
- Line 361: `get_submissions()` pagination - timeout=30
- Line 400: `get_assignment_name()` - timeout=30
- Line 493: Grade submission POST - timeout=60 (longer for large batches)
- Line 514: Progress polling - timeout=30
- Line 599: Course verification - timeout=30

### 2. Polling Loop Timeout ✅

**File**: `src/Programs/Autograder_Complete-Incomplete_v1-3.py`

Added 5-minute maximum wait time for grade job completion:
```python
max_wait_seconds = 300
start_time = time.time()
while True:
    if time.time() - start_time > max_wait_seconds:
        print(f"⚠️ Job timeout after {max_wait_seconds}s - assuming success")
        break
    # ... rest of polling logic ...
```

### 3. Improved Exception Handling ✅

**File**: `src/automation/canvas_helpers.py`

Changed bare `except:` to specific exception:
```python
except requests.exceptions.RequestException:
    break
```

Also added separate handling for Timeout vs ConnectionError in `get_assignment_groups()`.

### 4. Retry Logic with Exponential Backoff ✅

**File**: `src/automation/canvas_helpers.py`

Enhanced `_get_paginated()` with 3-attempt retry and exponential backoff:
```python
for attempt in range(3):
    try:
        response = requests.get(...)
        break  # Success
    except requests.exceptions.Timeout:
        wait_time = 2 ** attempt  # 1s, 2s
        time.sleep(wait_time)
```

### 5. Notification Timeout Optimization ✅

**File**: `src/automation/notification_manager.py`

Changed from single 30s timeout to split connect/read timeout:
```python
timeout=(5, 10)  # 5s connect, 10s read
```

### 6. Process Watchdog Service ✅

**Created**: `scripts/watchdog.sh` + `~/Library/LaunchAgents/com.autograder.watchdog.plist`

- Runs every hour at :30
- Kills any autograder process running > 2 hours
- Logs actions to `/Users/june/autograder_logs/watchdog.log`

### 7. Execution Time Tracking ✅

**File**: `src/automation/automation_engine.py`

Added timer to track and log total execution time:
```python
start_time = time.time()
# ... run automation ...
elapsed = time.time() - start_time
logger.info(f"⏱️  Total execution time: {minutes}m {seconds}s")
```

### 8. Pre-flight API Check ✅

**File**: `src/automation/automation_engine.py`

Added Canvas API connectivity check before processing:
```python
if not self.api.test_connection():
    logger.error("❌ Cannot connect to Canvas API - aborting run")
    return
```

Prevents wasting time if Canvas is down entirely.

## Expected Behavior Going Forward

### Normal Operations
- Automation runs daily at 2:00 AM
- Typical execution: 5-15 minutes for all courses
- Logs execution time at completion
- Watchdog checks hourly, kills any process > 2 hours old
- Logs rotate automatically when > 10MB

### Error Handling
- **Canvas timeout**: Retries 3 times with exponential backoff
- **Canvas completely down**: Aborts run with clear message
- **n8n not running**: Logs error but continues (non-critical)
- **Individual course fails**: Logs error, continues with other courses
- **Process hangs**: Watchdog kills it within 1 hour

### Warning Indicators
- Process running > 30 minutes (may indicate slow Canvas)
- Process running > 2 hours (will be killed by watchdog)
- No successful run in > 30 hours (check with `health_check.sh`)
