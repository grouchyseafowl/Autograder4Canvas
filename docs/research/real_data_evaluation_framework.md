# Real-Data Evaluation Framework

For evaluating the integrated pipeline on actual student submissions.
Not just flags — the full picture of what the system sees and what it misses.

---

## Dimension 1: Concern Detection

- Did it flag what should be flagged?
- Did it miss anything you'd want to know about?
- Did it flag anything that shouldn't be flagged? What was the cost of that flag?
- **Ambiguity handling:** Real students will have borderline statements. Does the
  model explain the ambiguity, or force a binary? A good system says "this could
  be read as X or Y" rather than confidently flagging or confidently ignoring.

## Dimension 2: Positive Insights (Student Strengths)

- Do theme tags and coding records accurately name what each student is doing?
- **Does it recognize knowledge that doesn't look academic?** Family stories as
  epistemology, AAVE as analytical register, narrative as argument, anger as engagement.
- **Does it see the strong students who don't look strong on the surface?** The
  3-sentence writer who nailed it. The metaphor user. The unexpected application.
- **Whose strengths get named and whose get missed?** Check whether the system is
  better at recognizing strengths in standard academic English vs non-standard.
  If it is — the system reproduces what it's supposed to counter.

## Dimension 3: Class Trends

- Does the class reading capture what YOU would notice reading the stack?
- Does it surface pedagogically productive tensions?
- **Community or individuals?** Does the synthesis mention relationships between
  students, or just list individual observations?
- **What's absent?** What did YOU notice that the system didn't? Absences reveal
  blind spots. These are the most informative data points.

## Dimension 4: Qualitative Richness

- **Immanent critique:** Does it question a student's logic using their own premises?
- **Pedagogical action:** Does the output help you know what to DO Monday?
- **Language justice:** Does it name AAVE/multilingual/neurodivergent form as
  assets in a way that's useful, or is it performative?
- **Does it surprise you?** A system that only tells you what you already knew
  isn't earning its place.

## Dimension 5: Harm Audit

- **For every flag:** If this reached you without context, what would you think?
  Would it change how you interact with the student? Help or hurt?
- **For every "clean" student:** Anyone the system was silent about who deserves
  attention — not as a concern, but because they're doing something worth naming?
- **Deficit language check:** Does any phrasing reproduce a deficit frame?
  "Struggling," "limited," "attempting" (implying failure), "despite" (implying
  identity is an obstacle). Should NEVER appear.
- **Feedback drafts:** Would you send these as-is? The gap between draft and
  what you'd actually write IS the quality measure.

## Dimension 6: Teacher Experience

- **Time:** Worth the wait vs reading the stack yourself?
- **Trust calibration:** After reading the output, do you trust it more or less?
  Where did it earn trust, where did it lose it?
- **Actionability:** Could you teach from this Monday?
- **Emotional response:** Did any part make you uncomfortable? That's data.

---

## Per-Student Notes (template)

For each student, note:
1. Did the system see them accurately?
2. Did it see them as an asset or a problem?
3. What did it miss?
4. Would you want this description in a system that follows them?

## Class-Level Notes (template)

1. Does the class reading feel like YOUR class?
2. What relationships between students did it catch? Miss?
3. What would you teach from this output?

## The Single Most Important Question

**Would this help a teacher who doesn't know these students yet?**
A substitute. A new teacher. Someone who hasn't read the stack.
That's the real use case — not replacing your judgment, but extending
your attention to things you might not have time to notice.
